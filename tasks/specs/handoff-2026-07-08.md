## 4. Locked product decisions (do not relitigate)

- **Auth:** Firebase Auth (Google + email/password), a **dedicated Firebase project** already
  created by leadership; invite-only provisioning; reset/invite mail via **Resend**. Pattern lifted
  from `divinelab/toulmin-lab`.
- **Data:** MongoDB, **same cluster, separate `admin` DB**. Congregant data never comingles with the
  public `website` DB.
- **Admin build order:** Foundation → **People & Calendar (M1b)** → Preaching Evaluator →
  Activity Orchestration. (People is the shared substrate both features hang off.)
- **Evaluator v1:** rubric-based — preacher self-score + mentor score/comments. AI-assist from the
  `/predica` transcript is **deferred**.
- **PWA + responsive nav:** installable; **bottom tab bar on mobile, sidebar on desktop**. This is
  an addition — the original `admin-design-prompt.md` mockups are sidebar-only.
- Bilingual admin UI (es-AR + en-US) from day one; granular per-feature RBAC with a permission
  matrix UI.

Sources: `tasks/specs/admin-platform-brief.md` (v0.2, approved), `admin-mvp.md` (CP1–CP6 with data
models, routes, edge cases), `admin-design-prompt.md`.

## 7. Conventions a fresh agent must know

- **Done is HUMAN-ONLY.** Agents own exactly two transitions: To Do→In Progress, In Progress→In
  Review. Never transition anything to Done, including duplicates — link + comment instead.
- `needs-refinement` label ⇒ **not** `/work`-ready. A **To Do** issue _without_ it is the ready signal.
- **Sensitive areas** always trigger the design gate: `email-services`, `form-pii-spam`,
  `likes-mongo`, `env-secrets`, `csp-headers`, `i18n-messages`, plus **auth/roles**.
- Contentful **model changes** go to the `staging` env only; alias cutover is human-only.
- Worktree-by-default; **never commit to `main`**. Conventional Commits, header ≤100 chars, and
  **body lines ≤100 chars** — commitlint _will_ reject a long unwrapped body line.
- Secret hygiene: reference env var **names** only, never values.
