## 5. How to actually run things

- **Run a migration** (CMA; targets `master-1.0.0`):
  ```bash
  cd <worktree>; set -a; source .env.local; set +a
  node scripts/contentful/run.mjs 01 --dry-run   # review the plan
  node scripts/contentful/run.mjs 01             # apply
  ```
  (`.env.local` supplies `CONTENTFUL_MANAGEMENT_ACCESS_TOKEN` + `CONTENTFUL_ENVIRONMENT=master-1.0.0`. A plain `node` does NOT auto-load `.env.local` — source it.)
- **Verify (env-independent, the commit gate):** `pnpm type-check && pnpm lint && pnpm test`.
- **Verify rendered:** `CONTENTFUL_ENVIRONMENT=master-1.0.0 pnpm build` (or `pnpm dev`); Next auto-loads `.env.local`. **`pnpm build` needs the Contentful tokens — only works because `.env.local` is in the worktree.** Clear `.next` when switching envs.
- **Contentful MCP for inspection:** the MCP's default `ENVIRONMENT_ID` still points at the now-deleted `agent-sandbox`, so either re-register it to `master-1.0.0` (`claude mcp add … -e ENVIRONMENT_ID=master-1.0.0 -e PROTECTED_ENVIRONMENTS=master,master-0.0.1` + full restart) **or pass an explicit `environmentId: "master-1.0.0"` on every MCP read**. Call `get_initial_context` first.
- **Commits:** Conventional Commits, header ≤100 chars; co-author trailer per the repo. Pre-commit runs a secret-guard + lint-staged prettier (it auto-formats `*.{json,md,css}` — expect prettier to reflow touched md/json).

## 6. Gotchas discovered during Task 0 (don't relearn these)

1. **Free-tier quota = 1 work env** beyond the `master` alias target (API: "Quota reached … 1 out of 1 allotted"). To create a new work env you must delete the existing one first — and **the safety classifier blocks env deletion** until the user explicitly authorizes that specific deletion.
2. **API keys are environment-scoped.** A newly created env returns `UNKNOWN_ENVIRONMENT` from the app until it's added to BOTH the Delivery + Preview keys (Settings → API keys → Environments). No MCP tool manages API keys → human action. _(Already done for `master-1.0.0`.)_
3. **Worktree starts with no `node_modules`** and **no `.env.local`** (gitignored, not copied on worktree creation). _(Both already fixed in this worktree.)_
4. `pnpm dev` **auto-enables draft mode** (`draftMode.ts`, `NODE_ENV==="development"`), so local reads via the **preview** token. Fine — the preview token now has `master-1.0.0` access.
5. **The migration runner** must be non-interactive (`yes: true`) or it crashes with `ERR_USE_AFTER_CLOSE` (no TTY). _(Already fixed.)_
6. **Production drifted since the audit:** a new **`sermon`** content type exists in prod (and thus in `master-1.0.0`). The epic doesn't touch it — leave it alone.
7. **`codegen.ts` was removed** from the repo (per the updated `CLAUDE.md`) — ignore the plan's optional note about aligning its env-var name; it's gone.
8. Code and model are **coupled**: the new code only works against `master-1.0.0`; production keeps working against `master` until cutover. So the branch's Vercel preview must get `CONTENTFUL_ENVIRONMENT=master-1.0.0` (branch-scoped) before preview QA — **set this at Task 12** when the PR/preview exists.
